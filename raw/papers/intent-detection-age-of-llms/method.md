# 方法详解：Intent Detection in the Age of LLMs

---

## 🏗️ 系统整体架构

论文提出三个核心方法，从简单到复杂逐步递进：

```
┌─────────────────────────────────────────────────────────────────┐
│                        方法一：SetFit 基线                        │
│                                                                  │
│  训练数据 ──→ 对比微调句子转换器 ──→ 分类头 ──→ 意图预测          │
│                      │                                           │
│                      └── 负面数据增强（关键词替换/删除）           │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                   方法二：自适应 ICL + CoT                        │
│                                                                  │
│  训练数据 ──→ Embedding ──→ 向量库                               │
│                                  │                               │
│  用户查询 ──→ Embedding ──→ 检索 Top-K 相似示例                  │
│                                  │                               │
│              意图描述 + 任务指令 ──→ 构建 Prompt ──→ LLM ──→ 意图 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                   方法三：混合系统（核心贡献）                      │
│                                                                  │
│  用户查询 ──→ SetFit (MC Dropout M 次)                          │
│                    │                                             │
│              预测方差 > 阈值？                                    │
│              ├── No ──→ 返回 SetFit 结果（快）                   │
│              └── Yes ──→ 调用 LLM 重新预测（准）                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 📦 组件一：SetFit 模型（Fine-Tuned Sentence Transformers）

### 两阶段训练

**阶段 1：对比微调**
- 基础模型：MPNet（all-mpnet-base-v2）
- 使用 Siamese 网络结构对句子对进行对比学习
- 目标：让同一意图的句子表示靠近，不同意图的远离

**阶段 2：分类头训练**
- 使用阶段 1 微调后的编码器生成句子表示
- 在表示上训练一个线性分类头（Sigmoid 激活，支持多标签）

### 超参数搜索空间

| 参数 | 搜索范围 |
|------|---------|
| body_learning_rate | 5e-6 ~ 5e-5 |
| head_learning_rate | 1e-3 ~ 1e-2 |
| num_epochs | 3 ~ 10 |
| batch_size | {8, 16, 32, 64} |
| n_trials | 10 |

使用 **Optuna** 进行超参数搜索。

### 负面数据增强（Negative Augmentation）

**目的**：帮助 SetFit 学习更好的决策边界。

**算法流程**：

```
输入: 训练集 D
输出: 增强后的训练集 D'

1. 对每个训练样本 x ∈ D:
   a. 用 KeyBERT 提取关键词 K = {k1, k2, ..., kn}
   b. 对每个关键词 ki:
      - 50% 概率: 完全删除 ki
      - 50% 概率: 替换为 5 字符随机字符串
   c. 生成增强样本 x_aug
   d. 标记 x_aug 为 OOS

2. 随机采样 |U| = 0.2 × |D| 个增强样本加入训练集
```

**示例**：

```
原始:   "looking for a gaming laptop"
         ↓ KeyBERT 提取关键词: [gaming, laptop]
增强 1: "looking for a"              (删除 "gaming laptop")
增强 2: "looking for a XYCVD QSDER"  (替换为随机串)
增强 3: "looking for a RTYUH"        (替换为随机串)
```

**为什么有效**：这些增强样本与原始样本有相似的词汇模式，迫使模型不能仅靠关键词"走捷径"。

## 📦 组件二：自适应 ICL + CoT（Adaptive In-Context Learning + Chain-of-Thought）

### 离线处理

```
1. 用句子转换器（BGE）编码所有训练样本 → 存入向量数据库
2. 用 LLM（Claude v3 Sonnet）为每个意图生成描述
```

### 在线推理

```
输入: 用户查询 q

1. 用 BGE 编码 q → 得到查询向量 v_q
2. 对每个意图 i:
   - 检索与 v_q 相似度 > t 的 Top-K 最相似训练样本
3. 构建 Prompt:
   ┌────────────────────────────────────┐
   │ [任务指令]                          │
   │ [意图描述列表]                      │
   │ [ICL 示例]                          │
   │ 用户查询: {q}                       │
   │ 请分析并预测意图。                   │
   └────────────────────────────────────┘
4. LLM 生成推理链（CoT）→ 输出意图预测
```

### 超参数搜索

| 参数 | 搜索范围 |
|------|---------|
| k (ICL 示例数) | {0, 1, 5, 10, 20} |
| t (检索阈值) | {1e-5, 0.3, 0.5, 0.7} |

**关键技巧**：
- **标签名随机化**：将意图名替换为 `Label-xx`，防止 LLM 依赖标签名的表面语义
- **ICL 示例按相似度降序排列**
- **标签顺序固定**：避免 LLM 对位置的偏好

## 📦 组件三：不确定性路由混合系统

### 核心思想

SetFit 做"快速通道"，LLM 做"慢速但精确的后备"。关键是**何时路由**。

### Monte Carlo Dropout 不确定性估计

```
输入: 用户查询 q

1. SetFit 模型开启 Dropout（p=0.1, hidden + attention layers）
2. 进行 M 次前向传播采样:
   for i = 1 to M:
       p_i = SetFit_with_dropout(q)  // 每次 Dropout 掩码不同
       label_i = argmax(p_i)

3. 计算不确定性:
   unique_labels = count_unique(label_1, ..., label_M)
   
   if unique_labels > 1 OR unique_labels < M/2:
       → 不确定，路由到 LLM
   else:
       → 确定，使用 SetFit 结果
```

### 路由决策示例

```
M = 10 次采样结果:
[Intent_A, Intent_A, Intent_A, Intent_B, Intent_A,
 Intent_A, Intent_A, Intent_A, Intent_A, Intent_A]

unique_labels = 2 > 1 → 不确定 → 路由到 LLM

对比:
[Intent_A, Intent_A, Intent_A, Intent_A, Intent_A,
 Intent_A, Intent_A, Intent_A, Intent_A, Intent_A]

unique_labels = 1, 且 1 < 10/2 → 不满足第二个条件也不满足第一个 → 确定
```

### 性能 vs 延迟权衡

增加 M 可以提高不确定性估计的精度，但实验表明 M 从 5 增到 20 的边际收益很小。M=10 是较好的平衡点。

**延迟分析**：
- 纯 LLM: 100% 延迟
- 混合 (M=10): ~43% 延迟（Claude Haiku）或 ~38%（Mistral Large）
- 若 MC 采样可批量进行，延迟还可进一步降低

## 📦 组件四：基于内部表示的 OOS 检测（两步法）

### 动机

控制实验发现 LLM 在以下条件下 OOS 检测能力严重退化：
- 意图标签范围变宽
- 标签空间变大

直接让 LLM 做"是否 OOS"的判断效果差，因为 LLM 在生成模式下倾向于输出已知标签而非拒绝。

### 方法

```
┌──────────────────────────────────────────────────────────┐
│                  两步 OOS 检测方法                         │
│                                                          │
│  离线阶段:                                                │
│  训练数据 ──→ LLM Decoder 最后一层 ──→ 提取表示           │
│             (最后一个 prompt token 的隐藏状态)              │
│             ──→ 存储每个训练样本的表示                     │
│                                                          │
│  推理阶段:                                                │
│  ┌───────────────────────────────────────────┐           │
│  │ Step 1: 纯 In-Scope 预测                    │           │
│  │                                           │           │
│  │ Prompt 中只包含 in-scope 标签（不含 OOS）  │           │
│  │ → LLM 预测最可能的 in-scope 意图           │           │
│  └─────────────────────┬─────────────────────┘           │
│                        ↓                                  │
│  ┌───────────────────────────────────────────┐           │
│  │ Step 2: 内部表示相似度比较                   │           │
│  │                                           │           │
│  │ 提取用户查询的 LLM 内部表示                 │           │
│  │ 与 Step 1 预测意图的训练样本表示比较         │           │
│  │ (余弦相似度 → 取均值)                       │           │
│  │                                           │           │
│  │ 相似度 > 阈值 → In-Scope                   │           │
│  │ 相似度 ≤ 阈值 → OOS                        │           │
│  └───────────────────────────────────────────┘           │
└──────────────────────────────────────────────────────────┘
```

### 为什么比直接让 LLM 判断 OOS 更好？

1. **简化任务**：Step 1 只做分类，不需要同时判断"选哪个"和"是否是 OOS"
2. **减少标签空间**：Step 2 只需比较与预测意图的相似度，而非所有意图
3. **内部表示更丰富**：编码了 LLM 对输入的"理解程度"，比生成文本更有信息量
4. **延迟开销小**：只需一次前向传播（~300ms），远快于自回归生成

### 关键实现细节

- 使用 Mistral-7B（开源权重，可提取内部表示）
- 取 decoder 层**最后一个 prompt token** 的表示（不是生成的 token）
- 余弦相似度比较
- 对预测意图的所有训练样本取**平均相似度**
- 阈值可调：平衡 OOS Recall 和 In-Scope Accuracy

## 🔬 控制实验设计

### 数据集构造

手动构建 20 个叶子意图，分属 2 个父类（Product Recommendation / Product Evaluation），每个叶子意图 10 条训练样本。

### 变量

- **S（意图范围）**：S ∈ {1, 2, 3, 4, 5}，随机组合 S 个叶子意图形成新意图
- **L（标签数量）**：L ∈ {1, 2, 3, 4, 5}，随机选择 L 个意图组成标签空间

### 实验设计

```
for S in [1, 2, 3, 4, 5]:      # 意图范围
    for L in [1, 2, 3, 4, 5]:  # 标签数量
        for trial in [1..10]:   # 10 次随机采样
            创建数据集 → 训练/评估 → 记录结果
```

### 核心发现

- **S 增大**：LLM OOS AUCROC 显著下降；SetFit 也下降但更平稳
- **L 增大**：LLM OOS AUCROC 严重退化；In-Scope 准确率也有下降但不如 OOS 严重
- **结论**：LLM 对 class design 的敏感度远高于传统模型

## ⚠️ 实现注意事项

1. **KeyBERT 选择**：用于负面增强的关键词提取，选择质量直接影响增强效果
2. **BGE 检索器**：使用 BAAI/bge-base-en-v1.5，需与训练数据编码保持一致
3. **MC Dropout 位置**：在 hidden layers 和 attention layers 都使用 dropout=0.1
4. **标签名掩码**：对开源数据集尤其重要，防止 LLM 从标签名推断语义
5. **标签顺序**：在所有实验中保持一致，避免位置偏差
