# 论文摘要：Intent Detection in the Age of LLMs

> EMNLP 2024 Industry Track | Amazon & IIT Jammu

---

## 🎯 研究背景

任务导向对话系统（TODS）是现代客户服务的技术核心。Gartner 预测到 2026 年，对话 AI 将为联络中心节省 800 亿美元的劳动力成本。**意图检测**是 TODS 自然语言理解层的关键组件，负责将用户查询映射到系统可执行的动作。

### 传统方法的痛点

| 痛点 | 说明 |
|------|------|
| **训练数据需求大** | 监督分类模型和句子编码器需要大量标注数据 |
| **OOS 检测差** | 无法有效识别超出支持范围的查询（OOS 查询空间无限大） |
| **数据不平衡** | 训练数据来自非 ML 专业的 bot 开发者，分布严重不均 |
| **少样本场景** | 每个意图可能只有很少的训练样本 |
| **意图范围差异大** | 从非常宽泛到非常具体的意图都有 |

---

## 📋 问题陈述

在 LLM 时代，如何在 TODS 中有效利用 LLM 进行意图检测？具体而言：

1. LLM 与传统微调模型在意图检测上的**性能/延迟权衡**如何？
2. 如何构建一个**兼具 LLM 准确率和低延迟**的可部署系统？
3. LLM 的 **OOS 检测能力**受哪些因素影响？
4. 能否利用 LLM 的**内部表示**改进 OOS 检测？

---

## 🏗️ 主要贡献

### 贡献 1：LLM vs SetFit 全面评估

- 在 3 个开源 + 3 个内部数据集上评估 **7 个 SOTA LLM**（Claude v1 Instant/v2/v3 Haiku/v3 Sonnet, Mistral 7B/Mixtral 8x7B/Mistral Large）
- 使用**自适应上下文学习（ICL）+ 思维链（CoT）提示**
- 与基于对比微调的句子转换器（SetFit）对比

### 贡献 2：混合系统（Hybrid System）

- 基于 **Monte Carlo Dropout 的不确定性路由**：SetFit 不确定时才调用 LLM
- **负面数据增强**：通过关键词替换/删除生成 OOS 训练样本
- 结果：F1 与 LLM 差距缩小到 ~2%，延迟降低 ~50%

### 贡献 3：OOS 检测控制实验

- 揭示 LLM 的 OOS 检测能力**受意图标签范围和标签空间大小显著影响**
- 对 class design 有重要指导意义

### 贡献 4：基于内部表示的 OOS 检测

- 利用 LLM decoder 层最后一个 prompt token 的表示
- 两步法：先预测 in-scope 标签 → 再用内部表示判断是否 OOS
- Mistral-7B 的 OOS 准确率和 F1 提升 >5%

---

## 📊 关键结果

### 核心性能对比（F1 Score 均值）

| 模型 | 平均 F1 | 平均延迟 (p50, s) |
|------|---------|-------------------|
| Claude v3 Haiku | **0.736** | 1.697 |
| Mistral Large | 0.735 | 3.565 |
| Claude v2 | 0.737 | 11.795 |
| SetFit + 负面增强 | 0.658 | **0.030** |
| SetFit (基线) | 0.600 | 0.030 |

### 混合系统效果

| 配置 | 平均 F1 | 延迟 (s) | 性能差距 | 延迟分数 |
|------|---------|----------|----------|----------|
| Claude v3 Haiku (纯 LLM) | 0.736 | 2.345 | 基准 | 1.000 |
| SetFit + Haiku (M=10) | 0.696 | 1.005 | -4.0% | 0.429 |
| Mistral Large (纯 LLM) | 0.735 | 3.867 | 基准 | 1.000 |
| SetFit + Mistral-L (M=10) | 0.696 | 1.453 | -3.8% | 0.376 |

### OOS 检测两步法（Mistral-7B）

| 数据集 | 基线准确率 | 改进后准确率 | 基线 OOS Recall | 改进后 OOS Recall |
|--------|-----------|-------------|----------------|-----------------|
| SOFMattress | 0.705 | **0.748** | 0.465 | **0.715** |
| Curekart | 0.601 | **0.761** | 0.376 | **0.782** |
| PowerPlay11 | 0.357 | **0.780** | 0.205 | **0.950** |

## 💡 一句话总结

**通过不确定性路由将 LLM 和轻量级 SetFit 结合，用 ~50% 的延迟达到 ~98% 的 LLM 准确率；同时揭示了 LLM OOS 检测的关键弱点，并提出了利用内部表示的有效改进方案。**
