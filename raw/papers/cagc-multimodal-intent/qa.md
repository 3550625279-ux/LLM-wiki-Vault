## 基础题（5 道）

### Q1: 什么是多模态意图识别 (MIR)？它与单模态意图识别有什么区别？

<details>
<summary>查看答案</summary>

**多模态意图识别 (MIR)** 是通过整合语言、视觉和听觉三种模态来理解人类意图的任务。

**与单模态的区别**：
- 单模态只使用一种信息源（如纯文本或纯图像）
- MIR 需要处理三种模态之间的**异质性**和**不一致性**
- MIR 面临**模态对齐**问题——不同模态的序列长度和特征维度不同
- MIR 可以利用**多模态互补**来提高意图理解的准确性

</details>

### Q2: CAGC 方法的两个核心创新是什么？

<details>
<summary>查看答案</summary>

**创新 1：Context-Augmented Transformer (CAT) 模块**
- 设计了跨视频上下文学习机制
- 通过 Cross-video Bank 筛选有效的跨视频源
- 使用渐进式注意力机制逐步引入跨视频上下文

**创新 2：Global Context-guided Contrastive Learning (GCCL) 方案**
- 引入全局上下文特征作为对比学习的监督信号
- 同时包含局部对比（mini-batch 内）和全局对比（跨整个数据集）
- 以文本模态为锚点，增强跨模态对齐

</details>

### Q3: Cross-video Bank 是如何构建的？为什么要两阶段？

<details>
<summary>查看答案</summary>

**两阶段构建**：

**Stage 1 — 初始构建**：
- 对每个视频计算视频级特征（帧均值池化）
- 用 L2 距离计算场景相似度
- 取 Top-k 最相似的视频作为跨视频集合 Ω(v_i)

**Stage 2 — 去噪**：
- 使用投票原则筛选意图倾向一致的视频
- 得到去噪后的集合 Ω*(v_i)

**为什么两阶段？**
- 仅靠场景相似度可能引入意图不相关的干扰视频
- 例如：两个场景相似的视频可能一个在讲笑话，一个在抱怨
- Stage 2 的意图一致性筛选确保了跨视频源的质量

</details>

### Q4: 为什么选择文本模态作为对比学习的锚点？

<details>
<summary>查看答案</summary>

**原因**：研究表明文本模态在意图识别中具有更大的重要性（Han et al., 2021）。

**设计逻辑**：
- 文本 = 最可靠的意图信号（语言直接表达意图）
- 视觉 = 可能有歧义（同样的表情可能表达不同意图）
- 听觉 = 辅助信号（语气可能不够明确）

**具体做法**：
- Anchor: 文本特征 h_t
- 正/负样本: 同/不同意图的视觉和听觉特征
- 全局对比: 文本特征 vs 跨视频的全局融合特征

</details>

### Q5: 训练损失函数包含哪几项？各自的权重是多少？

<details>
<summary>查看答案</summary>

**总损失**：
```
L_total = L_task + α(L_ta + L_tv) + β·L_g
```

**各项含义**：
- `L_task`: 意图分类交叉熵损失（主要任务损失）
- `L_ta`: 文本-听觉局部对比损失
- `L_tv`: 文本-视觉局部对比损失
- `L_g`: 全局对比损失

**权重**：
- α = 0.02（局部对比损失权重）
- β = 0.02（全局对比损失权重）
- 温度 τ = 0.7

**注意**：对比损失的权重很小（0.02），说明它们是辅助损失，主要任务还是分类损失。

</details>

## 中级题（5 道）

### Q6: CAT 中的"渐进式增强"具体是如何实现的？为什么比一次性融合更好？

<details>
<summary>查看答案</summary>

**渐进式实现**（Algorithm 1）：

```python
for j in range(1, k* + 1):
    # 逐步扩展 Key/Value
    K_c = h_c[1:j+1]  # 从1个跨视频逐步增加到k*个
    K, V = K_c
    M_j = softmax(Q @ K.T / sqrt(d)) @ V
    M_j = mean(M_j)
    M.append(M_j)
```

**为什么更好？**
1. **课程学习效果**：先学习最相关的上下文，再扩展到更多
2. **减少噪声累积**：每一步只引入少量新信息，避免一开始就受到干扰
3. **更好的梯度流**：渐进式结构提供了更平滑的梯度传播路径
4. **可解释性**：可以观察每一步引入的跨视频对结果的影响

</details>

### Q7: 全局对比学习 (GCCL) 与传统的 mini-batch 内对比学习有什么本质区别？

<details>
<summary>查看答案</summary>

**核心区别**：

| 维度 | Mini-batch 对比 | 全局对比 (GCCL) |
|------|----------------|-----------------|
| **对比范围** | 当前 mini-batch 内 | 整个训练数据集 |
| **正样本来源** | 同 batch 内意图相同的样本 | 跨视频中意图相同的全局特征 |
| **负样本来源** | 同 batch 内意图不同的样本 | 跨视频中意图不同的全局特征 |
| **监督信号** | 样本自身的多模态特征 | 包含跨视频上下文的全局特征 |
| **信息丰富度** | 有限（batch_size 个样本） | 更丰富（整个数据集） |

**本质区别**：GCCL 的监督信号（全局特征）包含了跨视频上下文信息，比单纯的样本特征更丰富、更鲁棒。

</details>

### Q8: 消融实验揭示了哪些关键发现？

<details>
<summary>查看答案</summary>

**消融实验结果**（MIntRec 20类 ACC）：

| 变体 | ACC | 性能下降 |
|------|-----|---------|
| 完整 CAGC | 73.39% | - |
| 去掉 CAT | 71.31% | -2.08% |
| 去掉 L_g | 72.22% | -1.17% |
| 去掉 L_l | 72.31% | -1.08% |
| 去掉 L_g 和 L_l | 71.66% | -1.73% |

**关键发现**：

1. **CAT 最重要**：去掉 CAT 性能下降最大（2.08%），说明跨视频上下文是核心
2. **全局对比比局部对比更重要**：去掉 L_g（-1.17%）比去掉 L_l（-1.08%）下降更多
3. **局部和全局互补**：同时去掉两者（-1.73%）比单独去掉任何一个都下降更多
4. **CAT + GCCL 协同**：只有两者结合才能达到最佳性能

</details>

### Q9: CAGC 在 MIntRec 和 CMU-MOSI 上的表现有什么差异？可能的原因是什么？

<details>
<summary>查看答案</summary>

**MIntRec（意图识别）**：
- CAGC 显著优于所有基线
- 20类 ACC: 73.39% vs 72.29% (MISA) → +1.10%
- 二分类 ACC: 90.11% vs 89.19% (MulT) → +0.92%

**CMU-MOSI（情感分析）**：
- CAGC 只在 ACC-2 和 F1 上略优于最佳基线
- ACC-2: 85.70% vs 85.52% (ConFEDE) → +0.18%
- 在 MAE、Corr、ACC-7 上不如其他方法

**可能原因**：
1. **任务差异**：意图识别比情感分析更依赖上下文（意图更模糊）
2. **数据集规模**：CMU-MOSI 较小（2199 样本），跨视频上下文的收益有限
3. **标签粒度**：CMU-MOSI 是连续情感分数，MIntRec 是离散意图类别
4. **跨视频场景多样性**：CMU-MOSI 的场景多样性可能不如 MIntRec

</details>

### Q10: Cross-video Bank 的去噪过程如何工作？它解决了什么问题？

<details>
<summary>查看答案</summary>

**去噪过程**：

1. 从 Stage 1 得到的 Ω(v_i) 中，筛选出意图倾向一致的视频
2. 使用**投票原则**：如果一个跨视频的意图与当前视频相同，则保留
3. 得到去噪后的 Ω*(v_i)，其中 1 ≤ k* ≤ k

**解决的问题**：

**问题 1：场景相似 ≠ 意图相似**
- 两个视频场景相似，但一个在讲笑话，一个在抱怨
- 直接使用场景相似的视频会引入误导性上下文

**问题 2：错误累积**
- 如果跨视频源质量差，CAT 学到的上下文特征会误导意图判断
- 去噪确保了跨视频源的质量，减少错误累积

**问题 3：干扰视频**
- 不相关的跨视频会引入噪声，干扰模型学习
- 去噪后的 Bank 只包含高质量的跨视频

</details>

## 高级题（5 道）

### Q11: CAGC 的计算复杂度主要受哪些因素影响？在大规模数据集上可能面临什么挑战？

<details>
<summary>查看答案</summary>

**计算复杂度分析**：

1. **Cross-video Bank 构建**：O(N² × d)
   - N 个视频，每对计算 L2 距离
   - 大规模数据集（如百万级）时成为瓶颈

2. **CAT 注意力计算**：O(N × k* × d²)
   - 每个视频需要与 k* 个跨视频计算注意力
   - 随 k* 和 d 线性/平方增长

3. **全局对比学习**：O(N × d)
   - 需要访问整个数据集的全局特征
   - 内存开销大

**大规模数据集的挑战**：

1. **Bank 存储**：需要存储所有视频的跨视频关系
2. **Bank 更新**：新数据加入时需要重新计算相似度
3. **全局对比的内存**：需要在 GPU 中存储整个数据集的特征
4. **Stage 2 的去噪**：需要遍历所有跨视频进行投票

**可能的解决方案**：
- 使用向量检索（如 FAISS）加速相似度搜索
- 使用近似最近邻（ANN）减少 Bank 构建时间
- 使用梯度累积或分布式训练处理内存问题
- 使用动量更新的 Bank（类似 MoCo）减少更新频率

</details>

### Q12: 如果要将 CAGC 应用到无监督场景（没有意图标签），需要做哪些修改？

<details>
<summary>查看答案</summary>

**核心挑战**：Stage 2 的去噪需要意图标签来判断意图一致性。

**可能的修改方案**：

**方案 1：自监督意图聚类**
- 用聚类算法（如 K-means）对视频特征进行聚类
- 将聚类结果作为伪意图标签
- 用伪标签进行 Stage 2 的去噪

**方案 2：去掉 Stage 2**
- 只使用 Stage 1 的场景相似度筛选
- 用更强的场景相似度阈值减少噪声
- 牺牲一些 Bank 质量换取无监督适用性

**方案 3：对比学习预训练**
- 先用自监督对比学习（如 SimCLR）预训练多模态编码器
- 用预训练的特征进行聚类得到伪标签
- 再用伪标签进行完整的 CAGC 训练

**方案 4：动量 Bank**
- 使用动量更新的 Bank（类似 MoCo）
- 在训练过程中逐步更新 Bank 中的特征
- 避免需要预计算的 Stage 2

</details>

### Q13: CAT 的渐进式注意力机制与标准 Transformer 的多头注意力有什么本质区别？

<details>
<summary>查看答案</summary>

**标准 Transformer 多头注意力**：
```
Q = h_v, K = h_c, V = h_c
Attention = softmax(Q·K^T / √d) · V
```
- 一次性使用所有跨视频的 K, V
- 所有跨视频具有相同的重要性权重

**CAT 渐进式注意力**：
```python
for j in range(1, k* + 1):
    K = h_c[1:j+1]  # 逐步扩展
    M_j = softmax(Q·K^T / √d) · V
    M_j = mean(M_j)
```
- 分 k* 步，每步只使用前 j 个跨视频
- 最终特征是所有步骤的累积

**本质区别**：

1. **信息引入方式**：标准 Transformer 一次性引入所有信息，CAT 渐进式引入
2. **注意力分配**：标准 Transformer 的注意力分布在整个跨视频集上，CAT 每步只关注部分
3. **特征融合**：CAT 通过 mean 操作和累积步骤，产生了更平滑的上下文表示
4. **噪声敏感性**：CAT 对噪声更鲁棒，因为每步只引入少量新信息

**类比**：
- 标准 Transformer = 一次性阅读所有参考文献
- CAT = 先读最相关的 1 篇，再读 2 篇，逐步扩展

</details>

### Q14: CAGC 的 Cross-video Bank 与 MoCo 的动量队列有什么异同？

<details>
<summary>查看答案</summary>

**相似点**：
1. 都是用于对比学习的记忆库
2. 都存储历史特征用于构建正负样本对
3. 都旨在扩展对比学习的范围

**不同点**：

| 维度 | MoCo 动量队列 | CAGC Cross-video Bank |
|------|--------------|----------------------|
| **存储内容** | 编码后的特征向量 | 视频索引 + 意图标签 |
| **更新方式** | 动量更新（每步更新） | 预计算（两阶段构建） |
| **筛选机制** | 无（FIFO 队列） | 场景相似度 + 意图一致性 |
| **用途** | 提供负样本 | 提供跨视频上下文 + 对比样本 |
| **是否需要标签** | 不需要（自监督） | Stage 2 需要（监督） |

**本质区别**：
- MoCo 的队列是**被动存储**：只是把历史特征存起来，不做筛选
- CAGC 的 Bank 是**主动筛选**：通过两阶段机制精心选择高质量的跨视频

**启发**：可以将 MoCo 的动量更新思想引入 CAGC，实现增量式的 Bank 更新，避免预计算的开销。

</details>

### Q15: 如果要在 CAGC 的基础上进一步提升性能，你会从哪些方向入手？

<details>
<summary>查看答案</summary>

**方向 1：改进 Cross-video Bank**
- 使用更鲁棒的场景相似度度量（如基于注意力的相似度）
- 引入层次化 Bank（粗粒度筛选 → 细粒度筛选）
- 使用动量更新避免预计算

**方向 2：增强 CAT**
- 引入多尺度注意力（不同层次的跨视频信息）
- 使用稀疏注意力减少计算开销
- 引入跨模态的上下文增强（不仅是视觉）

**方向 3：改进对比学习**
- 使用硬负样本挖掘（Hard Negative Mining）
- 引入课程学习策略（先简单后困难的对比样本）
- 使用多粒度对比（token 级 + 视频级）

**方向 4：扩展模态**
- 引入更多模态（如手势、身体姿态）
- 处理缺失模态的情况
- 模态特定的上下文增强

**方向 5：架构改进**
- 使用更强大的预训练模型（如 CLIP、GPT-4V）
- 引入图神经网络建模视频间关系
- 使用自适应权重调整 α、β

**方向 6：应用扩展**
- 应用到其他多模态任务（如情感分析、讽刺检测）
- 处理长视频（引入层次化 Bank）
- 实时推理优化

</details>
