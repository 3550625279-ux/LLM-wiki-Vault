# CAGC 方法详解

## 🏗️ 整体架构

CAGC 框架由四个核心组件构成，按数据流顺序排列：

```
┌──────────────────────────────────────────────────────────┐
│                    CAGC 整体架构                           │
│                                                          │
│  ┌─────────────┐    ┌──────────────────┐                 │
│  │  多模态编码器  │───→│  跨视频上下文学习   │                │
│  │  (Sec 3.2)   │    │  (Sec 3.3)       │                 │
│  └─────────────┘    └────────┬─────────┘                 │
│         │                    │                           │
│         ▼                    ▼                           │
│  ┌─────────────────────────────────┐                     │
│  │      多模态意图解码器 (Sec 3.4)    │                     │
│  │  ┌──────────┐  ┌──────────┐     │                     │
│  │  │ 局部融合  │  │ 全局融合  │     │                     │
│  │  │  (h_l)   │  │  (h_g)   │     │                     │
│  │  └────┬─────┘  └────┬─────┘     │                     │
│  │       └──────┬───────┘           │                     │

│  └─────────────────────────────────┘                     │
└──────────────────────────────────────────────────────────┘
```

## 📐 组件 1：多模态编码器 (Multimodality Encoder)

### 目标
将三种异构模态映射到统一的特征空间。

### 编码方式

| 模态 | 预训练模型 | 输出维度 (MIntRec) |
|------|-----------|-------------------|
| 语言 L | BERT | l × 768 |
| 视觉 V | ResNet | l × 256 |
| 听觉 A | wav2vec | l × 768 |

### 序列对齐

使用 **CTC (Connectionist Temporal Classification)** 模块将三种模态对齐到统一的词级序列：

```
h_t, h_a, h_v = SeqAligned(m_t, m_a, m_v)
# h_t, h_a, h_v ∈ R^{l × d}
# l = 统一序列长度, d = 统一特征维度
```

**关键点**：CTC 对齐解决了不同模态序列长度不一致的问题（文本30 tokens、视频230帧、音频180帧）。

## 🏦 组件 2：跨视频上下文学习

### 2.1 Cross-video Bank 构建

这是 CAGC 的**第一个核心创新**——一个精心筛选的跨视频记忆库。

#### Stage 1: 初始构建

**Step 1 — 视频级特征提取**：

对每个视频 v_i 的 F 个帧做均值池化：

```
v̄_i = (1/F) * Σ_{j=1}^{F} v_{frame(j)}^i
```

**Step 2 — 场景相似度计算**：

使用 L2 距离计算视频间的场景相似度：

```
score(v_i, v_c) = ||v_c - v_i||² = Σ_{t=1}^{T} (v_c^t - v_i^t)²
```

**Step 3 — Top-k 检索**：

取距离最小（最相似）的 k 个视频作为跨视频集合：

```
Ω(v_i) = {v_c^i(1), v_c^i(2), ..., v_c^i(k)}   // 按 L2 距离排序
```

#### Stage 2: 去噪过程

**问题**：仅靠场景相似度可能引入意图不相关的干扰视频。

**解决方案 — 投票原则**：
1. 从 Ω(v_i) 中筛选出与当前视频意图倾向一致的视频
2. 使用投票机制判断意图一致性
3. 得到去噪后的集合 Ω*(v_i)，其中 1 ≤ k* ≤ k

```
Ω*(v_i) = {v_c^i(1), ..., v_c^i(k*)}   // 只保留意图一致的
```

**直觉**：这就像在问"这些场景相似的视频中，哪些的说话者和当前视频有相似的意图？"

---

### 2.2 Context-Augmented Transformer (CAT)

**核心思想**：不是一次性把所有跨视频信息塞进去，而是**渐进式增强**上下文学习。

#### 算法流程（Algorithm 1）

```
输入: Q = h_v (当前视频视觉特征)
      K, V = h_c (跨视频视觉特征集合)
输出: h_m (长程上下文依赖特征)

M = []   // 存储累积的上下文特征

for j = 1, ..., k* do:
    # 渐进式扩展 Key/Value
    K_c = h_c^{1:j+1}   // 逐步加入更多跨视频
    K, V = K_c

    # 上下文增强注意力
    M_j = softmax(Q·K^T / √d) · V
    M_j = mean(M_j)     // 对跨视频维度取均值

    M.append(M_j)

h_m = M   // 拼接所有上下文层
```

**为什么渐进式？**
- 逐步引入跨视频信息，让模型先关注最相似的视频，再扩展到更多
- 避免一开始就受到不相关视频的干扰
- 类似于课程学习（Curriculum Learning）的思想

#### 特征融合

```
h_mv = LayerNorm(h_m + h_v)                    # 残差融合
h_cv = LayerNorm(Linear(h_mv) + h_mv)           # 非线性增强
```

这里 h_cv 就是捕获了长程上下文依赖的视觉特征。

## 🔗 组件 3：多模态意图解码器

### 局部多模态融合 (Local MF)

将文本与原始视觉/听觉特征融合：

```
# 门控融合
h_lf = Gate([h_t, h_v]) ⊙ Linear(h_v) + Gate([h_t, h_a]) ⊙ Linear(h_a)
h_l = NonLinear(h_t, h_lf) ⊙ h_lf
```

**门控机制**：让模型自适应地决定文本对视觉/听觉特征的关注程度。

### 全局多模态融合 (Global MF)

将文本与上下文增强的视觉/听觉特征融合：

```
h_gf = Gate([h_t, h_cv]) ⊙ Linear(h_cv) + Gate([h_t, h_ca]) ⊙ Linear(h_ca)
h_g = NonLinear(h_t, h_gf) ⊙ h_gf
```

**区别**：全局融合使用的是经过 CAT 处理的跨视频上下文特征 h_cv、h_ca，而非原始特征。

### 意图解码

```
h_out = TransformerDecoder(LayerNorm(h_l + h_g))
q = Linear(h_out)   # 分类 logits
```

## 📊 组件 4：全局上下文引导对比学习 (GCCL)

### 设计动机

| 对比方式 | 范围 | 局限 |
|---------|------|------|
| 传统方法 | Mini-batch 内 | 样本有限，对比信号弱 |
| CAGC | **整个数据集** | 通过全局特征扩展对比范围 |

### 以文本为锚点

**为什么选文本？** 研究表明文本模态在意图识别中具有更大的重要性。因此：
- **Anchor**: 文本特征 h_t
- **正样本**: 同意图标签的其他模态/全局特征
- **负样本**: 不同意图标签的其他模态/全局特征

### 局部对比损失

在 mini-batch 内构造正负样本对：

```
# 文本-听觉对比
L_ta = -Σ log[ Σ_{h_a⁺ ∈ P_l} exp(h_t · h_a⁺ / τ) / Σ_{h_a± ∈ P_l∪N_l} exp(h_t · h_a± / τ) ]

# 文本-视觉对比
L_tv = -Σ log[ Σ_{h_v⁺ ∈ P_l} exp(h_t · h_v⁺ / τ) / Σ_{h_v± ∈ P_l∪N_l} exp(h_t · h_v± / τ) ]
```

### 全局对比损失

跨整个数据集构造正负样本对：

```
# 锚点 h_t vs 全局融合特征 h_g
L_g = -Σ log[ Σ_{h_g⁺ ∈ P_g} exp(h_t · h_g⁺ / τ) / Σ_{h_g± ∈ P_g∪N_g} exp(h_t · h_g± / τ) ]
```

- **P_g**: 跨视频中意图标签相同的全局融合特征
- **N_g**: 跨视频中意图标签不同的全局融合特征

### 训练目标

```
L_total = L_task + α(L_ta + L_tv) + β·L_g
```

| 符号 | 含义 | 默认值 |
|------|------|--------|
| L_task | 意图分类交叉熵损失 | - |
| α | 局部对比损失权重 | 0.02 |
| β | 全局对比损失权重 | 0.02 |
| τ | 温度参数 | 0.7 |

## ⚠️ 实现注意事项

### Cross-video Bank 的维护时机
- **训练前**: Bank 在整个训练集上预计算（需要两阶段）
- **Bank 更新**: Stage 2 的去噪需要已训练的意图分类器
- **推理时**: Bank 也需要在验证/测试集上构建

### 计算复杂度
- CAT 的注意力计算涉及跨视频特征，复杂度为 O(N × k* × d²)
- 全局对比需要在整个数据集的全局特征上计算，存储和计算开销较大

### 超参数敏感性
- Top-k* 的选择直接影响跨视频质量（论文设为 7）
- 温度 τ 影响对比学习的分布锐度（论文设为 0.7）
- α、β 平衡分类和对比学习（论文设为相等的 0.02）
