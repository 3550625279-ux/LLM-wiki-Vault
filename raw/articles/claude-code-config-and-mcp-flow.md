 配置文件层级

  优先级从高到低：

  1. CLI 参数 (--mcp)        ← 临时，本次会话有效
  2. 项目 .mcp.json          ← 项目级，提交到 git 共享给团队
  3. 项目 settings.json      ← .claude/settings.json，项目级
  4. 本地 settings           ← .claude/settings.local.json，不提交
  5. 用户 .claude.json       ← ~/.claude.json，用户级（我们现在在这）
  6. 全局 settings.json      ← ~/.claude/settings.json，全局
  7. 托管/企业策略           ← 管理员下发的配置

  启动时发生了什么

  Claude Code 启动
    │
    ├─ 读取 ~/.claude/settings.json     → 加载 env、插件、主题
    ├─ 读取 ~/.claude.json              → 加载用户级 mcpServers、项目配置
    ├─ 读取 .claude/settings.json       → 加载项目级配置
    ├─ 读取 .mcp.json                   → 加载项目共享的 MCP
    │
    ├─ 合并所有 mcpServers              → 按优先级去重
    │
    └─ 对每个 MCP server：
         ├─ stdio 类型 → 启动子进程（npx ...）
         ├─ http 类型  → 建立 HTTP 连接
         └─ 验证连接   → claude mcp list 看到的 ✔/✘

  为什么改配置就够了

  MCP 服务器本身是独立运行的外部进程或远程服务。Claude Code 只需要知道：

  ┌────────────────┬────────────────────────┐
  │    配置字段    │ 告诉 Claude Code 什么  │
  ├────────────────┼────────────────────────┤
  │ type           │ 怎么连接（stdio/http） │
  ├────────────────┼────────────────────────┤
  │ command + args │ 启动什么命令（stdio）  │
  ├────────────────┼────────────────────────┤
  │ url            │ 连哪个地址（http）     │
  ├────────────────┼────────────────────────┤
  │ env            │ 传什么环境变量         │
  ├────────────────┼────────────────────────┤
  │ headers        │ 带什么认证头           │
  └────────────────┴────────────────────────┘

  所以：
  - 添加 MCP = 往配置文件里写一段 JSON
  - 删除 MCP = 从配置文件里删掉那段 JSON
  - 移动 MCP = 从一个配置文件剪切到另一个（项目级 → 用户级）
  - MCP 服务器本身不用动，它们是无状态的外部工具

  我实际做的事情

  第一次：claude mcp add           → 写入 ~/.claude.json 的 projects/C:/Users/35506/mcpServers
  第二次：claude mcp add -s user   → 写入 ~/.claude.json 的顶层 mcpServers
  第三次：node 删除项目级残留       → 清掉 projects 下的 mcpServers

  本质就是三次对同一个 JSON 文件不同位置的写操作。claude mcp add 只是个封装了 JSON 写入的 CLI 工具，底层和我用 node
  直接改文件是一样的。

  一句话总结

  MCP 管理 = 改 JSON 配置。Claude Code
  启动时读配置、按配置建立连接、把工具列表注入提示词。不需要"大规模动文件"，因为服务器是独立的，配置只是告诉 Claude Code
  "去哪里找它们"。

  四、底层协议细节

  MCP 协议基于 JSON-RPC 2.0

  // 请求格式
  {
    "jsonrpc": "2.0",
    "id": "<唯一ID>",
    "method": "<方法名>",
    "params": { ... }
  }

  // 响应格式
  {
    "jsonrpc": "2.0",
    "id": "<对应的请求ID>",
    "result": { ... }  // 成功
    // 或
    "error": { ... }   // 失败
  }

  常用 MCP 方法

  ┌────────────────┬─────────────────┬──────────────────┐
  │      方法      │      方向       │       说明       │
  ├────────────────┼─────────────────┼──────────────────┤
  │ initialize     │ Client → Server │ 初始化连接       │
  ├────────────────┼─────────────────┼──────────────────┤
  │ tools/list     │ Client → Server │ 获取可用工具列表 │
  ├────────────────┼─────────────────┼──────────────────┤
  │ tools/call     │ Client → Server │ 调用工具         │
  ├────────────────┼─────────────────┼──────────────────┤
  │ resources/list │ Client → Server │ 获取可用资源列表 │
  ├────────────────┼─────────────────┼──────────────────┤
  │ resources/read │ Client → Server │ 读取资源         │
  └────────────────┴─────────────────┴──────────────────┘

  stdio 通信方式

  ┌──────────────┐                    ┌──────────────┐
  │  Claude Code │  ──── stdin ────→  │  MCP Server  │
  │   (父进程)    │                    │   (子进程)    │
  │              │  ←── stdout ────   │              │
  │              │                    │              │
  │              │  ←── stderr ────   │  (日志/错误)  │
  └──────────────┘                    └──────────────┘

  用户: "搜索 Bocha MCP"
      │

  │ Claude Code (主模型)                                             │
  │                                                                 │
  │ 输入: 用户问题 + 系统提示 + 工具定义                               │
  │                                                                 │

  │ ┌─────────────────────────────────────────────────────────────┐ │
  │ │ {                                                           │ │
  │ │   "name": "bocha_search",                                   │ │
  │ │   "description": "使用 Bocha 搜索引擎搜索网页",               │ │
  │ │   "inputSchema": {                                          │ │
  │ │     "type": "object",                                       │ │
  │ │     "properties": {                                         │ │
  │ │       "query": { "type": "string" }                         │ │
  │ │     },                                                      │ │
  │ │     "required": ["query"]                                   │ │
  │ │   }                                                         │ │
  │ │ }                                                           │ │
  │ └─────────────────────────────────────────────────────────────┘ │
  │                                                                 │
  │ 模型决策: 需要调用 bocha_search，参数 query="Bocha MCP"          │
  └─────────────────────────────────────────────────────────────────┘
      │
      ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │ Claude Code (工具调度器)                                         │
  │                                                                 │
  │ 1. 识别工具来源: bocha (MCP 服务器)                               │
  │ 2. 构造 JSON-RPC 请求                                            │
  │ 3. 通过 stdin 发送给 bocha MCP 进程                              │
  └─────────────────────────────────────────────────────────────────┘
      │
      ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │ Bocha MCP 服务器 (Node.js 进程)                                  │
  │                                                                 │
  │ 1. 从 stdin 读取 JSON-RPC 请求                                   │
  │ 2. 解析: method="tools/call", name="bocha_search"               │
  │ 3. 执行: 调用 Bocha API                                          │
  │ 4. 构造 JSON-RPC 响应                                            │
  │ 5. 通过 stdout 返回给 Claude Code                                │
  └─────────────────────────────────────────────────────────────────┘
      │
      ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │ Claude Code (主模型，第二次调用)                                   │
  │                                                                 │
  │ 输入: 用户问题 + 工具调用结果                                      │
  │                                                                 │
  │ 模型决策: 整合搜索结果，生成最终回复                                │
  └─────────────────────────────────────────────────────────────────┘
      │
      ▼
  用户看到: "根据搜索结果，Bocha MCP 是..."
