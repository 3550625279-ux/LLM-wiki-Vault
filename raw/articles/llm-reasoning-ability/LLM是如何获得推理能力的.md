# LLM 是如何获得推理能力的？

> **来源**: 小红书「番茄的算法日记」
> **类型**: 技术科普长文（16张图文）
> **核心论点**: 大模型的推理能力不是预训练"涌现"的，而是通过后训练（post-training）一步步构建出来的。本文以鸡兔同笼问题贯穿，拆解从 SFT 到 PRM+PPO 再到 GRPO 的完整技术栈，展示 LLM 获得推理能力的过程。

---

## 1. 预训练给了什么，没给什么

预训练的目标只有一个：在海量语料上做 next-token prediction。

```
输入: 一个笼子里鸡和兔共有35个头、94只脚，
输出: 鸡

输入: 一个笼子里鸡和兔共有35个头、94只脚，鸡
输出: 和
```

这种自回归训练让模型在参数中编码了大量推理模式——数学课本的解题步骤、Stack Overflow 的推导过程、论文的证明链。但预训练模型面对问题时，倾向是**续写语料而非回答问题**：

```
输入: 一个笼子里鸡和兔共有35个头、94只脚，鸡和兔各有多少只？

预训练模型输出: 这是一个经典的数学问题，常见于小学奥数，类似的题目还有...
（续写语料，而不是解题）
```

**推理能力是潜藏的，未被激活的。后训练要做的就是激活它、定向它、强化它。**

## 2. 第一层：SFT —— 推理行为的定向激活

### 2.1 通用 SFT：先教模型"听懂指令"

教会模型从"续写模式"到"回答模式"。

最早的后训练实践（FLAN, Wei et al. 2021; InstructGPT, Ouyang et al. 2022）做的事很简单：用指令-回答 pair 做监督微调，让模型从"续写模式"切换到"回答模式"。

```
【输入（Prompt，不计算损失）】
请回答以下问题：一个笼子里鸡和兔共有35个头、94只脚，鸡和兔各有多少只？

【输出（Target，全量计算交叉熵损失）】
鸡有23只，兔有12只。
```

**对推理的贡献**：模型学会了"看到问题应该回答"。
**对推理的不足**：回答不含推理过程，直接给答案。换一道稍复杂的题，错误率暴增。

### 2.2 推理链 SFT：教模型"展开推理"

教会模型从"问题→答案"升级为"问题→推理链→答案"。

#### 2.2.1 推理链 CoT 数据

2022 年 Wei et al. 的 CoT 论文发现了一个关键现象：**让模型在回答中展示中间步骤，推理准确率从 17% 飙升到 78%。** 这个发现直接改变了 SFT 数据的构造方式——从"问题→答案"升级为"问题→推理链→答案"。

```
【输入（Prompt，不计算损失）】
请一步一步思考解决以下问题，最后输出答案。
一个笼子里鸡和兔共有35个头、94只脚，鸡和兔各有多少只？

【输出（Target，全量计算交叉熵损失）】
步骤1: 每只鸡2只脚，每只兔4只脚，所有动物只有1个头。
步骤2: 假设笼子里全是鸡，总脚数 35 × 2 = 70 只。
步骤3: 实际94只脚，多出 94 - 70 = 24 只。
步骤4: 每换一只兔多2只脚，兔 24 ÷ 2 = 12 只。
步骤5: 鸡 35 - 12 = 23 只。
最终答案: 鸡23只，兔12只。
```

#### 2.2.1 推理链 CoT 数据怎么获得？

推理链 SFT 的核心瓶颈是**数据从哪来**。人工标注推理链极其昂贵。实践中演化出了四条主流数据生产线：

**路径一：人工标注**

质量最高，但不可规模化。OpenAI 在训练早期推理模型时雇佣了大量标注员编写 CoT 数据。LIMA（Zhou et al., 2023）证明 1000 条精心标注的数据就能达到接近 GPT-4 的对话质量。在推理任务上规律更显著：一条清晰、正确、逻辑自洽的推理链，抵过十条只有答案没有过程的数据。

**路径二：教师模型蒸馏**

用更强的模型对海量问题生成推理链，作为小模型的 SFT 数据。这是目前开源社区最主流的数据生产方式。代表工作：

- **Self-Instruct**（Wang et al., 2023）：用少量种子指令让 LLM 自己生成更多指令-回答对
- **Evol-Instruct / WizardLM**（Xu et al., 2023）：从简单指令出发，让 LLM 通过"深化"和"拓宽"操作演化出复杂指令
- **Orca**（Mukherjee et al., 2023）：不仅蒸馏最终答案，还蒸馏教师的"解释轨迹"（explanation traces）——即教师在生成答案时的中间推理

**路径三：程序合成**

对于数学、编程等有明确求解器的领域，可以用程序自动生成问题+推理链+答案。例如用符号计算引擎生成代数题，用编译器验证代码正确性。这保证了 100% 的正确率，但覆盖的问题类型受限。

**路径四：Rejection Sampling**

让模型自己对同一问题生成 N 条推理链（如 N=64），用验证器筛出正确的，作为下一轮 SFT 的训练数据。即使模型初始正确率只有 30%，生成足够多样本后大概率能找到正确的。这是一种介于 SFT 和 RL 之间的数据策略——不涉及梯度更新，但利用了模型的采样能力。

> **重要性：质量 > 数量。** 在推理任务上，正确性是质量的第一维度——一条推理链再流畅、再多步骤，只要中间有一步错了，整条就是负样本。

### 2.4 SFT 的天花板

SFT 本质是行为克隆。模型学到的是"看到这类题，应该输出这种推理链"，而不是"为什么这样推理是对的"。主要有以下三个根本性瓶颈：

- **分布外泛化差**：训练数据全是代数推理→无法自发发明反证法
- **错误累积**：推理链超过 5 步，每一步的小误差会累积放大
- **无法自我纠错**：一旦推理方向错了，模型没有"回退重来"的机制

这三个瓶颈指向同一个结论：**仅靠模仿，无法超越被模仿者。** 需要一种能让模型自己"发现"更好推理方式的机制。

## 3. 第二层：RLHF —— 对齐偏好，但不针对推理

在进入推理专用强化学习之前，先看一条已经在产业界跑通的管线：InstructGPT 的 RLHF（Ouyang et al., 2022）。

### 3.1 三阶段管线

**阶段一：SFT**（同 2.1）

**阶段二：训练奖励模型（Reward Model, RM）**

给 RM 看同一个问题的两个回答，人类标注哪个更好，RM 学习预测这个偏好。注意：RM 的训练信号来自 pair-wise 比较（Bradley-Terry 模型），而非回归到绝对分数。

```
输入：问题 + 回答A + 回答B
  A：鸡有23只，兔有12只。
  B：鸡有20只，兔有15只。
人类标注：A > B

RM 训练目标：最大化 P(A > B) = σ(r(A) - r(B))
其中 σ 是 sigmoid 函数，r(·) 是 RM 输出的标量奖励
```

**阶段三：PPO 强化学习**

用 RM 作为奖励函数，优化策略模型。

PPO 的核心公式（简化）：

```
L_CLIP = E_t[min(r_t(θ) · Â_t, clip(r_t(θ), 1-ε, 1+ε) · Â_t)]

其中：
  r_t(θ) = π_θ(a_t|s_t) / π_old(a_t|s_t)  — 新旧策略的概率比
  Â_t = R - V(s_t)                           — 优势函数（实际奖励 - 基线）
  ε = 0.2                                     — clip 范围
```

直观理解：在保持每次更新幅度不过大（clip）的前提下，增大高奖励回答的概率，降低低奖励回答的概率。

### 3.2 RLHF 对推理的三重不足

1. **RM 不评估推理过程**：只看"回答好不好"，不看"推理对不对"。模型可能给出一个看起来合理但推理全错的回答，照样拿高分
2. **SFT 数据不含推理链**：InstructGPT 的 SFT 阶段，标注员直接写答案，不写推理过程
3. **奖励信号过于粗糙**：整个回答只有一个标量奖励，无法区分"答案对+推理对"和"答案蒙对+推理错"

## 4. 第三层：PRM+PPO —— 推理专用的强化学习

### 4.1 从结果奖励到过程奖励

**ORM（Outcome Reward Model）的问题：**

```
输入：问题 + 完整推理链 + 最终答案
输出：1 个标量

问题：推理过程全错但答案蒙对 → 高奖励
```

**PRM（Process Reward Model, Lightman et al., 2024）的核心改进：不对整条推理链打分，而是对每一步单独打分。**

```
【第1步输入】
问题：鸡和兔共35个头、94只脚...
推理前缀：步骤1：每只鸡2只脚，每只兔4只脚，所有动物1个头。
【第1步输出】得分：0.98（正确）

【第2步输入】
问题：同上
推理前缀：步骤1... 步骤2：假设全是鸡，35 × 2 = 70 只脚。
【第2步输出】得分：0.99（正确）

【错误步骤输入】
问题：同上
推理前缀：步骤1... 步骤2：假设全是鸡，35 × 4 = 140 只。
【错误步骤输出】得分：0.03（错误：混淆了鸡和兔的脚数）
```

关键：不是整条推理链只输入一次，而是到一个步骤的结束位置就输入一次，逐步骤打分。

**PRM 的架构**：基于 SFT 模型初始化，将最后一层的词表输出头替换为二分类头。

**PRM 的代价**：对一条 5 步推理链，需要 5 个逐步标注；ORM 只需要 1 个终局标注。标注量乘以推理链长度。

### 4.2 完整管线：SFT → PRM → PPO

**子步骤 1：策略模型生成多条推理链**

```
输入：鸡兔同笼问题
输出：同一问题采样多条不同推理路径

路径A（正确）：
  步骤1：设鸡 x 只，兔 y 只，x + y = 35
  步骤2：2x + 4y = 94
  步骤3：代入得 y = 12，x = 23
  答案：鸡23只，兔12只。

路径B（中间出错）：
  步骤1：设鸡 x 只，兔 y 只，x + y = 35
  步骤2：2x + 4y = 94，代入 2(35-y) + 4y = 94
  步骤3：70 - 2y + 4y = 94 → 2y = 14 → y = 7（错误：应为 2y = 24）
  答案：鸡28只，兔7只。
```

**子步骤 2：PRM 逐步骤打分**

```
路径A：步骤1=0.97 + 步骤2=0.98 + 步骤3=0.99 + 终局=1.0 = 总奖励 3.94
路径B：步骤1=0.97 + 步骤2=0.95 + 步骤3=0.05 + 终局=0.0 = 总奖励 1.97
```

**子步骤 3：PPO 参数更新**

高奖励路径（A）→ 优势 > n → 增大生成概率；低奖励路径（B）→ 优势 < n → 减小生成概率。clip 机制保证不会一次更新跳太远。

### 4.3 ORM+PPO 与 PRM+PPO 的两条子路径

| 维度 | ORM+PPO | PRM+PPO |
|------|---------|---------|
| **奖励粒度** | 只看最终答案 | 每一步单独打分 |
| **代表工作** | DeepSeekMath (Shao et al., 2024) | Let's Verify Step by Step (Lightman et al., 2024) |
| **优势** | 标注成本低，工程简单 | 能精确定位错误步骤 |
| **劣势** | 模型可能蒙对答案 | 标注成本极高 |

ORM+PPO 是推理强化的基础版，PRM+PPO 是精细化版。大多数实际系统会先用 ORM+PPO 跑通，再考虑升级到 PRM。

### 4.4 传统管线的局限

- **数据瓶颈**：SFT 推理链 + PRM 逐步标注，两者都极贵
- **奖励模型偏差**：PRM 本身是模型，可能对某些推理风格系统性地偏高/偏低打分
- **训练不稳定**：PPO 对超参敏感，KL 散度约束需要精心调参
- **天花板受限于 SFT**：SFT 模型推理上限低，RL 提升空间有限

## 5. 第四层：GRPO —— 绕开奖励模型的推理强化

DeepSeek-R1（2025）的最大突破是一个实验事实：**不经过任何 SFT，直接对基座模型做 RL，模型就能自发学会推理。** 这颠覆了"必须先 SFT 再 RL"的传统认知。

### 5.1 GRPO 的算法流程

传统管线（PRM+PPO）需要两个额外模型：PRM（过程奖励模型）和 Value Network（用于计算优势函数）。GRPO 全部绕开。

```
输入：鸡兔同笼问题

1. 对同一问题生成 G=16 条推理链：
  路径1：...答案：鸡23只，兔12只。✓
  路径2：...答案：鸡28只，兔7只。✗
  路径3：...答案：鸡23只，兔12只。✓
  ...
  路径16：...答案：鸡23只，兔12只。✓

2. 规则化奖励（无需任何训练模型）：
  正确性奖励：答案对 → 1.0，错 → 0.0
  格式奖励：输出了"最终答案：" → 1.0，否 → 0.0

  路径1 总奖励 = 1.0 + 1.0 = 2.0
  路径2 总奖励 = 0.0 + 1.0 = 1.0
  ...

3. 计算组内标准化优势（替代 Value Network）：
  μ = mean(所有路径的奖励)
  σ = std(所有路径的奖励)
  Â_i = (R_i - μ) / σ

4. 用 Â_i 替代 PPO 中的优势 A，进行策略梯度更新
```

### 5.2 GRPO vs PPO

| 维度 | PPO | GRPO |
|------|-----|------|
| **奖励来源** | PRM（需额外训练） | 规则化函数（无需训练） |
| **优势计算** | A = R - V(s)，需 Value Network | Â = (R-μ)/σ，组内相对排序 |
| **KL 约束** | 显式 KL 散度惩罚 | 有显式 KL 惩罚，但无需 Value Network |
| **适用前提** | 通用 | 需可验证答案（数学、编程） |

> **注意**：GRPO 仍有对参考策略的 KL 散度约束（防止策略偏离太远），"组内排序天然替代 KL"是一个常见误解。GRPO 简化的是优势计算和奖励获取，而非 KL 约束。

### 5.3 奖励 Hacking

GRPO 的规则化奖励看似简单，但实践中对奖励函数的设计高度敏感。几个关键经验：

**奖励 hacking 的常见形态**：

- **长度爆炸**：模型学会写超长推理链来"展示努力"，即使很多步骤是冗余或重复的
- **格式刷分**：模型学会在所有回答末尾机械地加"最终答案："来蹭格式分，但前面的推理质量没有提升
- **答案猜测**：模型对同一问题生成多条推理链时，每条链尝试不同答案，靠数量覆盖正确选项

**缓解策略**：

- 格式奖励只给到"有且仅有一次正确的格式输出"
- 引入长度惩罚（类似 RPO 的思路）或在奖励中显式减去 token 数量的惩罚项
- 对推理链增加多样性约束——同一问题的多条推理链之间应有足够差异

### 5.4 Aha Moment：推理能力的自发涌现

DeepSeek-R1-Zero（纯 RL，无 SFT）在训练中出现了论文称之为"Aha Moment"的行为。

```
【训练早期 step ~100】— 基本推理
  步骤1：设鸡 x 只，兔 y 只，x + y = 35，2x + 4y = 94
  步骤2：解方程得 y = 12，x = 23
  答案：鸡23只，兔12只。

【训练中期 step ~3000】— 涌现反思
  步骤1：设鸡 x 只，兔 y 只，x + y = 35，2x + 4y = 94
  步骤2：由第一个方程 x = 35 - y，代入 2(35-y) + 4y = 94
  步骤3：70 + 2y = 94，所以 2y = 24，y = 12
  等等，让我重新检查一下：
    方程 1：x + y = 35
    方程 2：2x + 4y = 94
    代入验证：2 × 23 + 4 × 12 = 46 + 48 = 94 ✓
    头数验证：23 + 12 = 35 ✓
  确认无误。答案：鸡23只，兔12只。
```

Aha Moment 更核心的特征不是末尾加一个验证，而是**模型在推理途中重新审视之前的步骤**——发现当前路径可能有问题，回溯到前面的步骤重新出发。"等等"之后的反思不仅是验证，有时还包括改变推理方向。

> 这些行为从未被人类标注过，从未出现在训练数据中。模型在"答案正确就得分"的简单奖励压力下，自己发现：验证 → 更高正确率 → 更多奖励。

### 5.5 R1 的完整训练流程

实际 R1 比 R1-Zero 更复杂，增加了冷启动 SFT 和多轮 RL 以稳定格式和保留通用能力：

```
DeepSeek-V3-Base
    ↓ GRPO（纯 RL，无 SFT）
R1-Zero（涌现推理，但格式不稳定：语言混杂、输出冗余）
    ↓ 用 R1-Zero 产出的推理链 + 通用 SFT 数据 做冷启动 SFT
SFT 模型（格式规范 + 推理保留）
    ↓ 第二轮 RL：推理任务用 GRPO + 通用任务用 RLHF
    ↓ 第二轮 SFT：Rejection Sampling 生成推理数据 + 通用能力数据
    ↓ 第三轮 RL：最终精调
R1（最终模型）
```

关键细节：第二轮 RL 不是纯粹的 GRPO——推理部分用 GRPO（规则化奖励），通用部分（写作、QA 等）仍用标准 RLHF（RM+PPO）。这解释了 R1 如何在获得推理能力的同时保持通用能力不退化。

## 7. 推理时的扩展：Test-Time Compute

前几层后训练都发生在训练阶段。2024 年以来，一个新的维度被系统化研究：**推理时投入更多计算。**

### 7.1 三种策略

**并行采样 + 多数投票（Self-Consistency, Wang et al., 2022）**

```
采样 N=5 次（temperature=0.7）：
  路径1：...答案：鸡23只，兔12只。
  路径2：...答案：鸡28只，兔7只。
  路径3：...答案：鸡23只，兔12只。
  路径4：...答案：鸡23只，兔12只。
  路径5：...答案：鸡19只，兔16只。

多数投票：23只鸡、12只兔（3次）> 其他
最终输出：鸡23只，兔12只。✓
```

**Best-of-N + Verifier**：不投票，而是用验证器（ORM/PRM）对 N 条推理链打分，选最高分的一条。与 Self-Consistency 的关键区别：不需要答案重复出现，只要验证器认为最好就选它。

**搜索 + 验证（MCTS / Beam Search）**

```
            [问题]
           /      \
      [步骤1a]    [步骤1b]
      /    \         |
  [步骤2a] [步骤2b] [步骤2c]
     |       |        |
  [✓答案]  [✗答案]  [✓答案]

PRM 对每个节点打分，低分路径剪枝
选择最高分路径
```

### 7.2 Test-Time Compute 的 Scaling Law

Snell et al.（2024, ICLR 2025）的实验结论：

- 推理时计算量 ×4 → 匹配参数量 ×14 的模型表现
- **收益递减**：计算量持续增加后，边际收益快速下降
- **策略分化**：简单任务适合并行采样，困难任务需要搜索+验证

## 8. 后训练的数据飞轮

后训练的各阶段并非孤立——它们之间存在**双向数据流动**。

### 8.1 迭代自训练

STaR（Zelikman et al., 2022）和 ReST（Gulcehre et al., 2023）提出了一个简洁的循环：

1. 模型对问题生成推理链
2. 筛选出推理正确且答案正确的样本
3. 用这些样本对模型再做 SFT
4. 更强的模型生成更高质量的推理链
5. 重复

这本质上是一个数据飞轮——RL 产出的高质量推理链反哺 SFT，SFT 后的模型又让 RL 有更高的起点。

### 8.2 大模型生产 + 小模型消费

DeepSeek-R1 的蒸馏实验揭示了一种分工模式：

```
大模型（数百B参数） → RL 训练 → 发现好的推理路径
                                    ↓
                           生成高质量推理链数据
                                    ↓
小模型（数B~数十B参数） → SFT → 习得推理能力（不自己做RL）
```

> 这意味着推理能力可能成为"基础设施"——由少数大模型生产，通过蒸馏分发给海量小模型。这一模式的经济学意义重大：**RL 训练的探索成本由大模型承担，小模型通过廉价的 SFT 获取推理能力。**

## 9. 推理能力的边界

**隐性跳步**：模型跳过它"觉得"显然但实际需要证明的中间步骤。
证明 √2 是无理数时，模型写"p² 是偶数，所以 p 是偶数"——跳过了"若 p 是奇数，p² 也为偶数"的反证法子证明。

**推理惯性**：一旦生成了一条推理链，模型倾向于"相信"它——即使后续步骤表明之前有误，也不会回退修正。SFT 模型最弱，RL 训练后有所改善（Aha Moment），但未完全解决。

**伪推理**：推理链语法正确，但基于虚假前提。模型会"编造"题目中不存在的数值条件，然后基于这些编造的条件推出一个看似合理的结论。对非专家读者最具迷惑性。

**规模门槛**：

```
5步逻辑推理链准确率：
  7B 模型  ~20%（几乎随机）
  70B 模型 ~55%（可用但不可靠）
  500B+ 模型 ~80%（相对可靠）
  RL 训练后 ~92%
```

## 10. LLM真的学会推理了吗？

如果"多想一会儿"就能提升准确率，那训练时学到的推理能力到底是什么？

个人有两种理解：

- **真实推理能力**：训练赋予生成合理推理链的能力；推理从中选出正确的那条推理链
- **模式匹配**：训练赋予推理的语法（推理链应该长什么样）；推理时搜索赋予推理的语义（找到真正正确的路径）

后一种理解如果成立，则意味着：**模型可能并不"理解"推理，它只是在搜索空间中找一条看起来正确的路径。**

事实上，两种理解都能找到证据。

- 支持"真实推理能力"：CoT 的多步衔接无法被纯模式匹配解释；RL 训练后涌现了训练数据中没有的推理路径；可验证任务上准确率持续提升超出预期。
- 支持"模式匹配"：换表述但保持逻辑 → 准确率波动；训练数据中未见的推理模式 → 几乎无法发明。

> 个人判断，LLM 推理能力或许介于纯模式匹配和纯逻辑推理之间，更接近**高效的模式补全**——在推理模式空间插值而非在逻辑空间演绎。但在足够大的规模和后训练下，这已经足够有用。

## 11. 全景图

```
预训练
  ↓ 语料中的推理模式 → 潜在推理能力
  ↓
后训练
  ├─ 第一层：SFT（行为激活）
  │   ├─ 通用 SFT：教模型"听懂指令"
  │   ├─ 推理链 SFT：教模型"展开推理"
  │   └─ 数据合成：Self-Instruct / Evol-Instruct / Orca / Rejection Sampling
  │       ↓
  ├─ 第二层：对齐方法群
  │   └─ RLHF（RM + PPO, Online）
  │   不足：不评估推理过程，只评估回答质量
  │       ↓
  ├─ 第三层：推理专用 RL
  │   ├─ ORM+PPO（DeepSeekMath）→ 只奖励最终正确
  │   └─ PRM+PPO（Let's Verify Step by Step）→ 逐步骤奖励
  │       ↓
  ├─ 第四层：纯 RL 推理
  │   └─ GRPO → 规则化奖励 → 推理能力涌现（DeepSeek-R1）
  │       ↓
  └─ 推理时：Test-Time Compute
      ├─ Self-Consistency（多数投票）
      └─ Best-of-N + Verifier

数据飞轮（贯穿所有层）
  ├─ STaR / ReST：迭代自训练
  ├─ 蒸馏：大模型生产、小模型消费
  └─ Rejection Sampling：采样筛选高质量数据

方法论横切面（影响所有层）
  ├─ 奖励设计：稀疏性、hacking 与缓解策略
  └─ 数据质量：正确性是推理数据的第一维度
```

## 12. 总结

大模型的推理能力不是一次技术突破的产物，而是后训练信号逐代升级的结果——从**"答案对不对"到"步骤对不对"再到"哪种推理策略更容易对"**。每往前一步，模型得到的反馈都更接近推理的本质。

沿着这条主线有三条并行发展的辅线：**数据生产线**（怎么高效地造推理训练数据）、**训练方法论**（Online RL 与 Offline DPO 的分野与融合）、**推理时优化**（test-time compute 的 scaling 与策略选择）。

下一步在哪里？当前研究沿三个主要路径推进，致力于让推理变得可验证、可调度、可执行：

**第一**，是更可靠的反馈机制。可验证奖励已经在数学、代码等任务上证明有效，但最终答案对了，不代表推理过程真的可靠。下一阶段需要把规则验证、工具执行、过程反馈和结果奖励结合起来。

**第二**，是推理预算的动态分配。强模型不应对所有问题都深度思考，而要学会判断什么时候快速回答，什么时候长链推理，什么时候调用工具，什么时候停止。

**第三**，是工具增强的代理式推理。复杂任务会越来越多地依赖"计划—执行—观察—修正—验证"的循环，推理能力不只是会想，还包括把中间判断放到外部环境中检验。

无论哪个方向，核心问题不变：**如何既能提升模型的推理能力，又能让推理过程更可验证、更可控？**
